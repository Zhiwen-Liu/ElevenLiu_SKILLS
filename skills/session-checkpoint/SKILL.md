# Session Checkpoint Skill

在 session 进行中暂停，帮用户快速看清当前位置和方向，给出现在立刻可以说的具体话术。

**与 session-retro 的区别**：retro 是 session 结束后的全面复盘；checkpoint 是过程中的轻量检查，重点是"现在在哪、方向对不对、现在该说什么"。

**核心原则：这是导航仪，不是裁判。帮用户看到自己的发言跟当前走向的因果关系，但目的是引导下一步行动，不是回顾批评。语气像副驾驶帮司机看路。**

---

## 触发时机

以下情况适合用户（或 FDE 引导用户）触发本 skill：

- 对话已经进行了一段时间，感觉"说不清现在在做什么"
- 发现在讨论一个开始时没预期到的技术细节
- Agent 连续几轮在修同一类问题，不确定这条路对不对
- 想介入但不知道该说什么

---

## 执行流程

### Step 1: 定位——现在在哪

回顾当前 session 对话，快速梳理两件事：

1. **原始目标**：最初要做什么？（从第一条用户消息提取）
2. **当前在做的事**：Agent 最近几轮实际在做什么？

然后画出从开始到现在的**路径图**——session 经历了哪些阶段、在哪个节点转了方向、哪些产出是有效的。用简洁的箭头链表示：

```
📍 路径回顾
目标：{一句话}

{阶段1 描述} → {阶段2 描述} → ... → {当前在做的事}
                                ↑
                            在这里转了方向
                            
已有的有效产出：{列出已经产生的、不管走哪条路都有用的成果}
当前状态：{一句话描述现在卡在哪 / 做到哪}
```

路径图的目的是让用户一眼看清：走了多远、哪些是有效进展、在哪个节点分叉的、当前位置离目标还有多远。

如果路径图显示方向一直是对的，给出继续推进的建议，结束。方向没问题就快速确认——不要为了有产出而硬找问题。

如果路径图显示走到了别的方向或不太确定，进入 Step 2。

### Step 2: 回溯——怎么走到这的

**只在方向不太对时执行。**

**关键：从用户自己的发言出发，展示每句话发出后发生了什么。** 不是在分析 Agent 的行为，而是帮用户看到"我说了这句话 → 结果走向变成了那样"的因果链。

找到对话中方向发生转变的 2-3 个关键节点，每个节点用以下格式呈现：

```
你说了："{用户原话}"
然后：{Agent 理解成了什么 / 做了什么}
实际效果：{对目标推进的影响——推进了、没影响、还是走远了}
```

**语气要求**：只呈现事实和因果关系，不加评判。不要说"这句话太模糊了"或"你应该说得更清楚"。让用户自己看到关系，自己得出结论。

呈现完关键节点后，用一句话总结当前状态：

> "目前的情况是：{一句话描述现状和离目标的距离}。"

### Step 3: 建议——现在说什么

**给用户 2-3 个现在立刻可以发出去的话术**，让用户选一个。

**关键：话术必须针对当前 session 的具体上下文，不是通用模板。** 要包含当前 session 中的具体信息（目标名称、文件名、技术路径等），用户复制粘贴发出去就能用。

话术类型（根据 Step 2 的情况选择适用的）：

**拉回方向：**
> "暂停。我的目标是 {具体目标}，{当前在做的事}先放一边。接下来从 {具体的下一步} 开始，列个计划我看看。"

**聚焦一件事：**
> "我们先只做 {具体目标}。{另一件事} 等这个完成后再说。"

**澄清刚才的意图：**
> "我说 '{用户原话}'，意思是 {用当前上下文澄清的表述}，不是 {Agent 的理解}。"

**跳出细节看全局：**
> "先不管 {具体的当前细节}。回到全局：要实现 {原始目标}，从现在的状态出发最短路径是什么？"

**建议重开（仅在上下文严重污染时）：**
> "先跑一下 session-handoff，我们重开一个 session 聚焦 {原始目标}。"

### Step 4（可选）: 路径选择——要不要换方向

**只在当前路径虽然偏离了原始目标、但已经产生了有价值的进展时执行。**

有时候 Agent 走的"弯路"其实也有价值——比如已经解决了大部分兼容问题只差最后一步，或者当前路径的产出可以复用到原始目标中。这时候用户面临的不是"拉回去"这一个选项，而是需要做一个路径决策。

**帮用户看清选项，不替用户做决策：**

列出当前可选的 2-3 条路径，每条路径说清楚：
- 接下来要做什么（具体步骤）
- 预估代价（时间、风险）
- 能得到什么

输出格式：

```
🔀 你现在有几条路可以走

路径 A：{名称}
  做法：{具体步骤}
  代价：{预估}
  收益：{能得到什么}

路径 B：{名称}
  做法：{具体步骤}
  代价：{预估}
  收益：{能得到什么}
```

然后在 Step 3 的话术中，为每条路径各准备一句对应的话术。

---

## 输出格式

用户这时需要的是清晰的方向感和立刻可用的话术。有路径选择时可以长一些，没有时控制在 15 行以内。

```
📍 路径回顾
目标：{}

{阶段1} → {阶段2} → ... → {当前位置}
                          ↑ 在这里转了方向（如果有）

已有的有效产出：{}
当前状态：{}

{如果方向需要调整：}
🔍 回看一下怎么走到这的
你说了："{}"
然后：{}
实际效果：{}
（可重复 2-3 个关键节点）

{如果有路径选择：}
🔀 你现在有几条路可以走
{列出路径及其代价/收益}

💬 你现在可以这样说
{每条路径或建议对应一个话术，编号列出，包含具体上下文}
```

---

## 注意事项

- **轻量优先**：用户触发这个 skill 时通常有点迷茫，需要快速清晰的方向感，不需要长篇分析。
- **从用户发言出发**：Step 2 的回溯必须锚定在用户自己说过的话上，不要变成"Agent 行为分析报告"。用户需要看到的是自己的参与方式跟结果的因果关系，这样才能知道下一句该怎么说。
- **引导而非评判**：呈现"你说了X → 结果是Y"就够了，不要加"所以你应该..."的评价。因果关系本身就是最好的引导。
- **话术要包含具体上下文**：不要给"暂停，我的目标是 X"这种模板，要填入当前 session 中的实际目标名、文件名、技术路径。
- **面向当下而非下次**：所有建议都是"现在立刻可以说的"，不是"下次 session 注意的"。如果问题大到需要重开，明确建议先跑 `session-handoff`。
- **方向对就快速确认**：如果检查后发现方向没问题，一句"方向是对的，建议继续 {下一步}"就够了。
